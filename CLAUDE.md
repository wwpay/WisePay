# WisePay 프로젝트 규칙

## 파일 수정 시 필수 규칙

### 수정 로그 기록

**어떤 파일을 수정하든** 반드시 아래를 실행할 것:

1. **수정한 파일 자체**의 상단(주석 처리 가능한 위치)에 수정 이력을 추가한다
2. **index.html**은 수정 여부와 관계없이 반드시 기록한다

### 기록 형식

```
수정: YYYY-MM-DD HH:MM — [수정 내용 간단히]
```

예시:
```
수정: 2026-05-22 19:50 — freee 급여 CSV 업로드 UI 추가 + GAS importPayrolls 핸들러
```

### index.html 기록 위치

`index.html`의 `<head>` 안 최상단 주석 블록에 기재:

```html
<!--
  수정: 2026-05-23 10:00 — app.js: 급여 계산 로직 수정
-->
```

이전 수정 내용은 삭제하고 **최신 1줄만** 유지한다.

### 반드시 지킬 것

- 처음 기동 시 Claude Code CLI를 최신 버전으로 전역 업데이트하고 시작하는데 혹시 안 되면 그냥 skip.
- 파일을 한 줄이라도 수정했으면 반드시 이 규칙을 적용한다
- index.html은 수정하지 않았더라도 **반드시 업데이트**한다
- 날짜/시간은 실제 수정 시각으로 기재 (모를 경우 사용자에게 확인)
- 날짜/시간은 일본 현재의 시간이며, 반드시 **JST(일본 표준시, Asia/Tokyo)** 기준으로 기록한다
- 정확한 시간은 시스템(노트북 등) 또는 인터넷을 통해 정보를 취득한다.
- 코드에서 날짜/시간을 다룰 때는 `Utilities.formatDate(new Date(), "Asia/Tokyo", "yyyy-MM-dd HH:mm:ss")` 사용
- 내용은 한국어로 작성
- 수정이 끝나면 무조건 커밋&푸쉬할 것

## 수정 완료 후 출력 형식

수정이 완료되면 반드시 아래 형식으로 결과를 출력한다:

```
### 요구사항
[사용자가 요청한 내용을 그대로 또는 요약하여 기재]

### 수정 내용
[변경된 파일과 변경 내용을 구체적으로 기재]
```

예시:
```
### 요구사항
임금대장 인쇄 시 스크롤바가 출력된다 — 숨김 처리해 줘.

### 수정 내용
- css/styles.css: @media print에 .annual-scroll-wrap::-webkit-scrollbar { display:none } 추가
- js/app.js: beforeprint 이벤트에서 overflow:visible 인라인 스타일 적용
```
